# 『数独 × バラトロライク（仮）』総合プロトタイプ仕様書

## 1. プロジェクト概要
* **コンセプト:** 「数独の配置ルール」×「手札のマネジメント」×「ライン消去」。プレイヤーはランダムに配られる数字（1〜9）を手札とし、数独のルールに従って9x9の盤面に配置していくデッキ構築ローグライク。
* **戦略の核:** 盤面が詰まる前に「列」や「ブロック」を完成させて盤面を掃除（クリア）し、巨大な乗算スコアを叩き出す。
* **デザイン指針:** 「サイバー・ストイック」。黒背景に、白とネオンブルーの細いグリッド線、そして発光する数字（1〜9）のみ。思考の邪魔になる一切のエフェクトを排除する。

## 2. ゲームサイクルと進行
盤面と山札を継続して使い回す、シームレスなサバイバル進行です。

1. **ブラインド開始:** 目標スコアが提示される。山札（初期デッキは1〜9が各5枚ずつの計45枚など）から、手札に「5枚」の数字が配られる。
2. **パズルフェーズ:** プレイヤーは手札から数字を1つ選び、数独のルールを満たす「空きマス」に配置（Play）してスコアを稼ぐ。
3. **ドローとディスカード:** 配置した分だけ山札から補充される。配置できるマスがない場合、回数制限付きの「ディスカード（手札交換）」を使って危機を回避する。
4. **目標到達と精算:** 目標スコアに到達すれば「ROUND CLEAR」ボタンが出現（オーバーラン可能）。精算するとショップへ移行する。
5. **次ラウンドへ:** **盤面の状態と山札の残り**を引き継いで次へ進む。山札が尽きた場合は、捨て札と盤面から消去された数字をシャッフルして山札を再構築する。

## 3. コアメカニクス

### 3.1 盤面と配置ルール（数独の絶対法）
* **グリッド:** 9 × 9マス（3×3のブロックが9つ）。
* **配置ルール:** 縦の列（9マス）、横の列（9マス）、およびそのマスが属する3×3のブロック内に、**「手札と同じ数字」が既に存在している場合は配置できない**。
* **ライン消去（掃除）:** 縦1列、横1列、または3x3ブロックのいずれかの9マスが「すべて埋まった（1〜9が揃った）」瞬間、その9マスの数字は盤面から消滅（クリア）し、空きマスに戻る。

### 3.2 ゲームオーバー条件（詰み）
手札にあるどの数字も盤面に配置できず（数独のルール違反になる）、かつ「ディスカード（手札交換）」の残り回数も「0」になった場合、手詰まりとなりゲームオーバー（Run終了）。

### 3.3 スコア計算ロジック
数字を1つ盤面に配置するごとに計算が走ります。
`獲得スコア = (配置した数字の価値 + チップ加算) × (ベース倍率 + 消去ボーナス) × (アーティファクト乗算)`

* **チップ（基本点）:** 配置した数字そのものが基本チップになる（例：「9」を置けば 9チップ、「1」なら 1チップ）。
* **ベース倍率:** 初期値 `×1.0`。
* **消去ボーナス（大インフレの引き金）:**
    * 数字を置いたことで「ライン消去」が発生した場合、ベース倍率に `+5.0`。
    * 「縦と横」「列とブロック」など、2つ以上のエリアが同時に消去（クロス消去）された場合、最終倍率が `×3` される。

## 4. ローグライク要素（デッキ構築と強化）

### 4.1 アーティファクト（パッシブ装備 / 最大5枠）
「どの数字をデッキに増やすか」というシナジーを生み出します。

| カテゴリ | 名前 | 効果 |
| :--- | :--- | :--- |
| **チップ系** | **奇数の哲学** | 手札から「1, 3, 5, 7, 9」を配置した時、チップに `+20`。 |
| **倍率系** | **コーナー・ピース** | 盤面の四隅（4マス）のいずれかに数字を配置した時、倍率 `+3.0`。 |
| **条件乗算** | **クロスワード** | クロス消去（縦・横などの同時消去）が発生した時、最終倍率が `×3` から `×5` に強化される。 |
| **成長系** | **ナンバー・ナイン** | 「9」を配置してライン消去を発生させるたび、このアーティファクトの倍率が永続的に `+0.5` される。 |
| **ルール介入** | **妥協案** | 各3x3ブロックにおいて、1つだけなら「同じ数字が重複しても配置できる」ようになる（数独のルール破壊）。 |

### 4.2 コンシューマブル（使い切り・タロット枠）
手札の事故や盤面の詰まりを解消するアイテムです。

| 名前 | 効果 | UX |
| :--- | :--- | :--- |
| **消しゴム** | 盤面に配置されている任意の数字を1つ選んで消去する（空きマスにする）。 | 対象の数字がノイズと共にフェードアウト。 |
| **インクの書き換え** | 手札にある数字1枚を選択し、好きな数字（1〜9）に変化させる。 | 選択後、1〜9のミニキーパッドUIが出て選択。 |
| **強制執行** | 次の1回のみ、数独の配置ルール（重複禁止）を完全に無視して、手札をどこにでも配置できる。 | 盤面全体が赤く明滅し、警告音と共に制限が解除される。 |
| **増刷** | 手札にある数字1枚を選択し、デッキ（山札）にその数字のコピーを「3枚」追加する。（※デッキを特定の数字に偏らせるための必須アイテム） |

### 5.1 UX・操作のポイント
* **配置アシスト:** 手札の数字をクリック（またはドラッグ開始）した瞬間、盤面上の**「数独のルール上、配置不可能なマス」が暗くグレーアウト**する。これにより、プレイヤーはいちいち頭で全列の重複を確認するストレスから解放され、テンポ良くプレイできる。
* **消去エフェクト:** 列やブロックが揃った瞬間、その9マスがネオンブルーに強く発光し、スッと消滅する。この時スコアの倍率（Mult）が跳ね上がるアニメーションを同期させる。

## 6. 開発ロードマップ（JavaScript）

* **Phase 1: 数独盤面とバリデーションの構築**
    * 9x9の二次元配列とUIグリッドの作成。
    * 「縦・横・ブロック」の重複チェック関数の実装（手札を選んだ時の配置可能マス判定）。
* **Phase 2: ドロー＆プレイサイクルの実装**
    * 1〜9の配列（デッキ）を作成し、シャッフルして5枚を手札（Hand）に描画する処理。
    * マスへの配置、手札の消費、山札からの自動ドロー、ディスカード機能の実装。
* **Phase 3: ライン消去とバラトロスコア計算**
    * 配置時に「縦・横・ブロック」が9つ埋まったかを判定し、該当マスを空にする処理（Tetris的な消去）。
    * スコア計算（チップ×倍率）と、消去ボーナスの実装。
* **Phase 4: ローグライク拡張（ショップ・アイテム）**
    * ラウンド目標の判定、ショップ画面への遷移。
    * アーティファクトによる計算式のHook（介入）や、コンシューマブルによる盤面・手札操作の実装。
