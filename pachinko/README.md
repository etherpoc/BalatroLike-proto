# 『パチンコ × バラトロライク（仮）』総合プロトタイプ仕様書

## 1. プロジェクト概要
* **コンセプト:** 「重力と物理演算」×「デッキ構築ローグライク」。落下するボールがピン（釘）にぶつかるたびにスコア計算式が育ち、最下部のポケットに入ることでスコアが爆発する、重力落下型のスコアアタック。
* **戦略の核:** 盤面のピンを強化（デッキ構築）し、アーティファクト（パッシブ）で物理挙動や計算式をバグらせる。
* **デザイン指針:** 「サイバー・ストイック」。暗闇の中に発光する「点（ピン）」と「光の軌跡（ボール）」のみ。パチンコ特有の過剰な演出やノイズは一切排除し、純粋な物理挙動と数字のインフレに没頭させる。
* **開発環境:** ブラウザ（HTML5 / JavaScript）。物理演算には軽量な2D物理エンジン（Matter.js など）の使用を推奨。

## 2. ゲームサイクル
1. **ブラインド開始:** 目標スコアが提示される。プレイヤーには限られた「ボール（手数）」が与えられる。
2. **ドロップフェーズ:** 盤面上部の任意の横位置から、ボールを1球投下する（Play）。
3. **落下＆計算:** ボールがピンに衝突するたびに、リアルタイムで計算式（チップや倍率）が加算されていく。
4. **ポケットイン（精算）:** 盤面最下部にある複数の「ポケット」のいずれかにボールが入ると、最終的な計算が行われスコアが確定する。
5. **ラウンドクリア:** ボールが尽きる前に目標スコアに到達すればクリア。ショップフェーズへ。
6. **次ラウンドへ:** **盤面のピンの配置と強化状態を引き継いで**次のラウンドへ進む。

## 3. コアメカニクス

### 3.1 盤面（ボード）と物理要素
* **盤面:** 縦長の長方形。無数の「ピン（釘）」が規則的、または不規則に配置されている。
* **ピンの種類（初期）:**
    * **青ピン（チップ）:** 当たるたびにチップ `+10`。
    * **赤ピン（倍率）:** 当たるたびにベース倍率 `+1.0`。
* **ポケット（最下部）:** 横に分割されたゴール地点。入った場所によって最終ベース値が異なる。
    * *中央:* 高難易度。基本チップ `+50` / 倍率 `+3.0`
    * *両端:* 低難易度。基本チップ `+10` / 倍率 `+1.0`

### 3.2 スコア計算ロジック
ボールが落下し、最下部のポケットに入った瞬間に以下の式で最終スコアが確定します。
`獲得スコア = (ポケット基本チップ + 衝突したピンのチップ合計) × (ポケット基本倍率 + 衝突したピンの倍率合計) × (アーティファクト乗算)`

* **リアルタイムUI:** ボールが跳ねるたびに、画面横の `[チップ] × [倍率]` の数字が「チャリン」という音と共にリアルタイムに増えていくことで、落下中の期待感を煽ります。

## 4. ローグライク要素（盤面構築と強化）

### 4.1 アーティファクト（パッシブ装備 / 最大5枠）
物理挙動とスコア計算の両方に介入するジョーカー枠です。

| カテゴリ | 名前 | 効果 |
| :--- | :--- | :--- |
| **物理介入系** | **低重力空間** | ボールの落下速度が下がり、バウンドしやすくなる。（※ピンに当たる回数が劇的に増える） |
| **物理介入系** | **バンパー** | 盤面のランダムなピン3本が「弾き返すピン（バンパー）」になり、当たるとボールが上に跳ね上がる。 |
| **乗算系** | **ピンボール・ウィザード** | 1回の落下で「30回」以上ピンに衝突した場合、最終倍率が `×3` になる。 |
| **条件系** | **ウォール・バウンド** | ボールが左右の「壁」に衝突するたびに、倍率が `+2.0` される。 |
| **ギャンブル** | **ブラックホール** | 最下部のポケットのうち1つが「倍率 ×0（スコア没収）」になる代わりに、他のすべてのポケットの倍率が `×2` になる。 |

### 4.2 コンシューマブル（使い切り・タロット枠）
盤面のピン（デッキ）を直接強化・改造するアイテムです。

| 名前 | 効果 | UX |
| :--- | :--- | :--- |
| **ピンの打ち込み** | 盤面の何もない「空きスペース」をクリックし、新しいピンを1本追加する。 | 光の点が新しく灯る。 |
| **属性付与（赤）** | 選択した青ピン（チップ）を、赤ピン（倍率）に書き換える。 | 色がネオンレッドに変化。 |
| **ガラスのピン** | 選択したピンを「ガラス化」する。当たると倍率が `×2` になるが、30%の確率で砕けて盤面から消滅する。 | ピンが白く透過発光する。 |
| **磁力化** | 選択したピンを「マグネット」にする。近くを通るボールをわずかに引き寄せる。 |

## 6. JavaScript実装ロードマップ

物理演算をゼロから書くのは円の衝突判定（円と円、円と線）が複雑になりやすいため、「Matter.js」などの既存ライブラリを導入すると非常にスムーズに開発できます。

* **Phase 1: 物理演算ボードの構築 (Matter.js導入)**
    * 重力空間の定義と、静的剛体（Static Body）としてのピンの配列描画。
    * クリックしたX座標から動的剛体（Dynamic Body）のボールを落下させる処理。
    * ボールとピンの衝突検知（Collision Event）。
* **Phase 2: スコア計算のバインディング**
    * ボールがピンに衝突した際、ピンの属性（青/赤）を取得し、UI上のチップ/倍率の変数を加算する処理。
    * 最下部にセンサー（見えない壁）を配置し、通過したポケットに応じた最終計算とボールの消去処理。
* **Phase 3: ラウンド進行とアイテムシステム**
    * 目標スコア判定と、ボール数の管理。
    * コンシューマブルアイテムによる、特定のピンの属性変更（Bodyのプロパティ書き換え）。
* **Phase 4: アーティファクトの実装**
    * 物理エンジン自体に介入するアーティファクト（重力スケールの変更、反発係数 restitution の変更）の実装。
    * スコアの最終計算時に割り込むフック処理の実装。
